# ws-relay

Минималистичный WebSocket-сервер-релей для связи между клиентами по токену. Сервер не обрабатывает и не хранит полезную нагрузку, а только маршрутизирует сообщения между подключёнными участниками.

## Концепция

Основная идея проекта — предоставить простой и прозрачный слой для двустороннего обмена данными между двумя типами участников: `agent` и `client`.

-   **`agent`** — участник, который принимает входящие команды и отправляет результаты их выполнения.
-   **`client`** — участник, который отправляет команды и ожидает результаты.

Каждое соединение идентифицируется по **токену**. Сервер пересылает сообщения только между участниками с одинаковым токеном, создавая таким образом изолированные каналы связи.

## Ключевые особенности

-   **Маршрутизация по токену**: Сообщения доставляются только в пределах одного токена.
-   **Двунаправленная связь**: Данные передаются в обоих направлениях: `client` -> `agent(s)` и `agent` -> `client(s)`.
-   **Поддержка нескольких участников**: К одному токену может быть подключено несколько агентов и несколько клиентов. Сообщение от одного участника будет доставлено всем подходящим получателям.
-   **Прозрачность**: Сервер не читает и не изменяет содержимое сообщений.
-   **Простое логирование**: Ключевые события (подключение, отключение, пересылка) выводятся в консоль.
-   **Обратная связь**: Если клиент отправляет сообщение, но ни один агент не подключён, клиент получит сообщение об ошибке.

## Как это работает

Клиент должен подключиться к серверу, указав в URL два GET-параметра: `token` и `type`.

**Формат URL:**
`ws://<host>:<port>?token=YOUR_TOKEN&type=YOUR_TYPE`

-   `token` — любая строка, идентифицирующая канал связи.
-   `type` — тип участника, может быть `agent` или `client`.

## Установка

Для работы проекта требуется Node.js и npm.

1.  Клонируйте репозиторий или скачайте файлы проекта.
2.  Откройте терминал в корневой папке проекта и установите зависимости:

```sh
npm install
```

## Запуск сервера

Для запуска сервера выполните команду:

```sh
npm start
```

По умолчанию сервер запустится на порту `11169`. Вы можете изменить порт, отредактировав переменную `port` в файле `server.js`.

В консоли вы увидите сообщение об успешном запуске:
`[INFO] WebSocket relay server started on port 11169`

## Тестирование

В проекте есть встроенная страница для интерактивного тестирования.

1.  Убедитесь, что сервер запущен.
2.  Откройте файл `test.html` в вашем браузере.
3.  На странице доступно 4 панели (2 агента и 2 клиента) для симуляции различных сценариев.
4.  Используйте разные или одинаковые токены, подключайтесь и отправляйте сообщения, чтобы проверить логику маршрутизации.
