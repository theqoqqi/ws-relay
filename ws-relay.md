# Техническая спецификация для интеграции с WebSocket-релеем `ws-relay`

## 1. Общее описание

`ws-relay` — это минималистичный WebSocket-сервер, который действует как релей (ретранслятор). Его задача — маршрутизировать сообщения между двумя типами участников (`agent` и `client`) внутри изолированных каналов, идентифицируемых по токену. Сервер является **прозрачным** (payload-agnostic), он не читает, не изменяет и не хранит содержимое пересылаемых сообщений.

## 2. Модель участников

Существует два типа участников, которые необходимо указать при подключении:

-   **`agent`**: Агент. Его роль — получать сообщения от клиентов, обрабатывать их и, опционально, отвечать.
-   **`client`**: Клиент. Его роль — отправлять сообщения агентам и, опционально, получать ответ.

## 3. Протокол подключения

Подключение осуществляется по стандартному протоколу WebSocket (`ws://`).

**URL для подключения:**
`ws://<host>:<port>?token=YOUR_TOKEN&type=YOUR_TYPE`

**Параметры (обязательные):**

| Параметр | Тип      | Описание                                                              | Пример        |
| :------- | :------- |:----------------------------------------------------------------------| :------------ |
| `token`  | `string` | Уникальный идентификатор канала связи.                                | `my-secret-A` |
| `type`   | `string` | Тип участника. Допустимые значения: **`agent`** или **`client`**.     | `agent`       |

**Важно:** Если один из параметров отсутствует или `type` имеет неверное значение, сервер немедленно закроет соединение.

## 4. Правила маршрутизации сообщений

Логика пересылки сообщений строго определена и зависит от типа отправителя.

-   **Когда `client` отправляет сообщение:**
    -   Оно будет доставлено **всем** `agent`, которые в данный момент подключены с **тем же `token`**.
    -   Оно **не** будет доставлено другим `client`.

-   **Когда `agent` отправляет сообщение:**
    -   Оно будет доставлено **всем** `client`, которые в данный момент подключены с **тем же `token`**.
    -   Оно **не** будет доставлено другим `agent`.

## 5. Обработка ошибок и особые случаи

Код, использующий `ws-relay`, должен быть готов к следующим ситуациям:

-   **Отправка от `client` при отсутствии `agent`:**
    -   Если `client` отправляет сообщение, но ни один `agent` не подключён, сообщение просто игнорируется. `client` **не получит** никакого уведомления об этом.

-   **Отправка от `agent` при отсутствии `client`:**
    -   Если `agent` отправляет сообщение, но ни один `client` не подключён, сообщение просто игнорируется. `agent` **не получит** никакого уведомления об этом.

-   **Потеря соединения:**
    -   Если соединение с сервером обрывается (проблемы с сетью, перезапуск сервера), клиентский код должен уметь обнаруживать это (через событие `onclose` сокета) и, при необходимости, выполнять логику переподключения.
